# xTools

xTools 是一款 macOS 优先的本地效率工具，目标是提供紧凑型全局启动器、文本与文件输入识别、轻量超级面板和内置功能指令系统，帮助用户快速启动应用、打开文件、执行计算、处理文本和调用常用本地动作。

> 当前项目处于 MVP 规划阶段，尚未开始正式实现。详细实施方案见 [xTools macOS MVP 实施计划](./xtools-macos-mvp-plan.md)。

## 产品定位

xTools 希望成为一个轻量、快速、可靠的桌面效率入口。它不追求首版就复刻完整插件市场，而是优先把最常用的本地工作流做好：

- 用一个全局输入框快速找到应用、文件、URL 和本地命令。
- 根据输入内容自动匹配可执行动作。
- 对选中的文本或文件弹出上下文动作面板。
- 用统一的功能指令系统组织内置能力，为后续插件化扩展预留接口。

## MVP 功能范围

### 全局启动器

- 默认通过 `Option + Space` 唤起。
- 采用紧凑型浮窗设计，类似 Spotlight/uTools 的轻量输入框。
- 支持输入文本、粘贴文本、拖入文件或文件夹。
- 支持键盘选择结果，并通过 `Enter` 执行动作。

### 多类型输入处理

- 支持文本输入。
- 支持文件和文件夹输入。
- 支持从剪贴板或拖拽内容生成输入上下文。
- MVP 暂不包含图片、截图、OCR 和翻译输入链路。

### 超级面板

- 默认通过 `Option + Shift + Space` 唤起。
- 可选择启用鼠标中键触发。
- 根据当前选中的文本、文件或剪贴板内容展示可用动作。
- 鼠标触发和选中内容捕获可能需要 macOS 辅助功能权限。

### 功能指令系统

- 使用内置 `Command Registry` 管理所有功能指令。
- 支持按输入类型过滤可用动作。
- 支持关键词、模糊匹配、拼音和首字母匹配。
- 首版只实现内置指令，不开放第三方插件加载和插件市场。

## 首版内置指令

- **应用启动：** 扫描并启动 macOS 应用。
- **文件/文件夹打开：** 索引常用目录并用系统默认方式打开。
- **URL 打开：** 识别 URL 或常见域名并用默认浏览器打开。
- **基础计算器：** 支持四则运算、括号、小数和运算优先级。
- **文本动作：** 去除首尾空白、转大写、转小写、复制结果。
- **文件动作：** 打开、在 Finder 中显示、复制路径。

## 技术栈

- [Tauri](https://tauri.app/)：桌面应用外壳和系统能力集成。
- Rust：全局快捷键、窗口控制、剪贴板、文件和应用操作。
- React：启动器、超级面板和设置页界面。
- TypeScript：前端类型约束和功能指令接口。
- Vite：前端开发和构建工具。
- `fuse.js`：模糊搜索。
- `pinyin-pro`：中文拼音和首字母匹配。

## 默认快捷键

| 功能 | 快捷键 |
| --- | --- |
| 显示/隐藏全局启动器 | `Option + Space` |
| 打开超级面板 | `Option + Shift + Space` |
| 执行选中指令 | `Enter` |
| 关闭当前浮窗 | `Esc` |

## 暂不包含

以下能力不进入 MVP，将作为后续阶段单独规划：

- 翻译。
- AI 能力。
- OCR。
- 截图输入。
- 第三方插件加载。
- 插件市场。
- MCP 能力暴露。
- 全盘搜索。
- Windows/Linux 支持。
